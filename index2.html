CSS样式表的三种引入方式

CSS（层叠样式表）是用于设置网页样式的语言，它可以通过三种不同的方式引入到HTML文档中。每种方式都有其适用场景和优缺点。

方式一：内联样式

内联样式是直接在HTML标签中使用style属性来定义CSS样式。这种方式的优先级最高，但不利于样式的复用和维护。

<div style="color: blue; font-size: 18px; margin: 20px;">
    这是使用内联样式的文本
</div>

优点：

  • 优先级最高，可以覆盖其他方式定义的样式
  • 直观明了，样式与元素直接关联
  • 适用于需要特殊样式的单个元素

缺点：

  • 代码冗余，不利于维护
  • 样式无法复用
  • HTML结构与样式混合，违背了"内容与表现分离"的原则

方式二：内部样式表

内部样式表是在HTML文档的head部分使用style标签定义的CSS样式。这种方式适用于单个页面的样式定义。

<head>
    <style>
        /* CSS样式规则 */
        body {
            font-family: Arial, sans-serif;
            background-color: #f0f0f0;
        }
        
        h1 {
            color: navy;
            text-align: center;
        }
    </style>
</head>

优点：

  • 样式可以在页面内复用
  • 不需要额外的文件请求
  • 适用于样式较少的单个页面

缺点：

  • 样式无法在多个页面之间共享
  • 增加了HTML文件的大小
  • 不利于浏览器缓存

方式三：外部样式表

外部样式表是将CSS代码保存在独立的.css文件中，然后在HTML文档中使用link标签引入。这是最常用的方式，有利于样式的复用和维护。


<head>
    <link rel="stylesheet" href="styles.css">
</head>

/* styles.css文件中 */
body {
    font-family: Arial, sans-serif;
    background-color: #f0f0f0;
}

h1 {
    color: navy;
    text-align: center;
}

优点：

  • 样式可以在多个页面之间共享
  • 实现了内容与表现的分离
  • 便于维护和更新
  • 浏览器可以缓存CSS文件，提高加载速度

缺点：

  • 需要额外的HTTP请求加载CSS文件
  • 对于简单的页面可能显得过于复杂

CSS优先级比较三种引入方式的优先级顺序

当同一个元素被多种方式定义的样式影响时，CSS会按照一定的优先级规则来决定最终应用哪种样式：

  1. 内联样式：优先级最高
  2. 内部样式表/外部样式表：优先级相同，取决于它们在文档中的顺序，后定义的会覆盖先定义的
  3. 浏览器默认样式：优先级最低

此外，CSS选择器本身也有优先级，从高到低依次是：

  1. !important 声明
  2. 内联样式
  3. ID选择器
  4. 类选择器、属性选择器、伪类选择器
  5. 元素选择器、伪元素选择器
  6. 通配符选择器
/* 优先级示例 */
<p id="special" class="text" style="color: red;">这段文本是什么颜色的？</p>

/* CSS */
#special {
    color: blue;
}

.text {
    color: green;
}

p {
    color: purple;
}

在上面的例子中，文本最终会显示为红色，因为内联样式的优先级最高。
